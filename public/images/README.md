# คู่มือการจัดเก็บและตั้งชื่อไฟล์รูปภาพโครงการ (Sutra Construction)

เพื่อความสะดวก ตรวจสอบง่าย และป้องกันการแคชภาพ (Caching) ของเซิร์ฟเวอร์ เช่น Vercel/CDN ต่อไปนี้คือโครงสร้างระบบจัดเก็บรูปภาพของโครงการทั้งหมดที่สอดรับกับซอร์สโค้ดในระบบครับ

---

## 📂 1. โครงสร้างโฟลเดอร์รูปภาพ (`/public/images/`)

ระบบแบ่งสัดส่วนรูปภาพออกเป็น 5 หมวดหมู่หลัก ดังนี้:

```text
/public/images/
├── house-designs/          # แบบบ้านมาตรฐาน (แบ่งตามสไตล์ดีไซน์)
│   ├── modern/             # 1. สไตล์โมเดิร์น
│   │   ├── Sutra-modern1/  # แบบบ้าน ศุทราโมเดิร์น ๑
│   │   ├── Sutra-modern2/  # แบบบ้าน ศุทราโมเดิร์น ๒
│   │   └── Sutra-modern3/  # แบบบ้าน ศุทราโมเดิร์น ๓
│   ├── contemporary/       # 2. สไตล์คอนเทมโพลารี่ (ร่วมสมัย)
│   ├── western/            # 3. สไตล์ตะวันตก / นอร์ดิก
│   ├── thai-applied/       # 4. สไตล์ไทยประยุกต์ / ล้านนา
│   ├── modern-classic/     # 5. สไตล์โมเดิร์นคลาสสิก
│   └── muji/               # 6. สไตล์มูจิ (อบอุ่นสไตล์ญี่ปุ่น)
│
├── ready-houses/           # โครงสร้างบ้านพร้อมอยู่ / ทาวน์โฮม
│   ├── c-home/             # โครงการ C-Home Project
│   ├── p-place/            # โครงการ P-Place Project
│   └── luxury/             # โครงการศุทรา Luxury
│
├── lands/                  # รายการที่ดินแบ่งขายเปล่าทำเลทอง
│   ├── land-1.jpg          # สันกำแพง

```

## 🏷️ 2. กฎการตั้งชื่อและการจัดเก็บแบบยั่งยืน

### กฎข้อที่ 1: แยกโฟลเดอร์เฉพาะแบบบ้านทุกแบบ (Subfolder per Model)
* ไม่เก็บไฟล์แบบรวมกระจุกอยู่ที่จุดเดียว 
* รูปภาพทั้งหมดของแบบบ้านแบบหนึ่งๆ (เช่น **ศุทราโมเดิร์น ๑**) จะต้องอยู่ภายใต้โฟลเดอร์เฉพาะของตนเอง เช่น `/public/images/house-designs/modern/Sutra-modern1/`
* ไฟล์ภาพหน้าปกให้ตั้งชื่อให้จำง่าย เช่น `main.jpg` หรือ `Sutramodern001a.jpg` และรูปภาพในแกลเลอรีเสริมก็แยกย่อยในหมวดหมู่เดียวกัน

### กฎข้อที่ 2: ป้องกันปัญหาภาพเก่าไม่ยอมอัปเดตบน Vercel (Cache Busting)
Vercel และเบราว์เซอร์ของผู้ใช้อาจใช้ระบบ **Edge CDN Cache** เพื่อโหลดหน้าเว็บได้เร็วขึ้น ทำให้เมื่อเราทับภาพเดิมด้วยชื่อเก่า (เช่น `cover.jpg` ตัวใหม่ ทับ `cover.jpg` ตัวเก่า) ระบบอาจยังดึงภาพเก่ามาแสดง
* **ทางเลือกระยะยาว (ดีที่สุด):** ใส่เวอร์ชันต่อท้ายตัวย่อในไฟล์ภาพ เช่น `Sutramodern001a_v2.jpg` และเข้ามาเปลี่ยนสิทธิ์ลิงก์ในโค้ด `App.tsx` เพื่อให้ CDN รับรู้ว่าเป็นไฟล์ใหม่ทันที
* **ทางเลือกอย่างง่าย:** เติม Query Parameter ท้าย URL ภาพในซอร์สโค้ด เช่น:
  `/images/house-designs/contemporary/Sutra-contemporary1/Sutracontempolary001a.jpg?v=2`
  การเติม `?v=2` หรือรันหมายเลขไปเรื่อยๆ จะบังคับให้เซิร์ฟเวอร์ Vercel ไปดึงภาพใหม่ล่าสุดจากเครื่องของคุณทันทีโดยไม่ต้องลบแคช

## 🛠️ 3. วิธีการอัปเดตรูปลงระบบและซอร์สโค้ด

เมื่อต้องการอัปโหลดรูปภาพใหม่หรือแต่งภาพในระบบ:
1. เตรียมไฟล์ภาพเป็นประเภท `.jpg` หรือ `.png` ที่บีบอัดขนาดไฟล์แล้ว (แนะนำไม่เกิน 300KB - 500KB เพื่อความรวดเร็วในการเปิดหน้าเว็บ)
2. วางภาพลงในโฟลเดอร์ที่ถูกต้องตามพิกัดโครงสร้างด้านบน
3. ตรวจสอบโค้ดใน `src/App.tsx` ที่มีการอ้างอิงถึงเส้นทางรูปภาพนั้น 
4. ตรวจทานและแก้ไขเส้นทางชื่อไฟล์ (รวมถึงใส่ `?v=...` หลังนามสกุลไฟล์เพื่อเคลียร์แคชถ้าจำเป็น)
5. ทำการ Push หรือ Sync ไปยัง GitHub เพื่อให้ Vercel ทำการแปลง Build ระบบและเป็นแบบเรียนรู้โดยอัตโนมัติ
